# Summary — 2026-04-30

**Branch:** `feature/login_signup`

## Changes

### `src/app/features/login/login.component.ts`

Redesigned the login page from a simple PrimeNG card form into a **split-panel sliding card** UI.

- **Removed** PrimeNG dependencies (`ButtonModule`, `InputTextModule`, `CardModule`, `PasswordModule`) — replaced with plain HTML inputs and buttons styled via a large component-scoped `styles: []` block.
- **New layout:** Two panels side by side inside a fixed 768×480 card. A blue overlay panel slides left/right (CSS `transform + transition`) to reveal/hide the sign-in and sign-up forms. Controlled by `isSignUpActive: boolean` toggled via `toggleMode()`.
- **Sign-in form:** Fully functional — bound to `username`/`password` via `ngModel`, calls `AuthService.login()` on submit, shows error banner and loading state.
- **Sign-up form:** UI-only — fields are unbound, submit button has no handler. No registration API exists yet.
- **Social login buttons:** Decorative only (G+, f, GitHub, LinkedIn).
- **Demo credentials** hint rendered in the UI (`admin`/`admin123`, `user`/`user123`).

### `sketch/` (new directory)

Added HTML/CSS/JS prototype files (`preview.html`, `style.css`, `script.js`) and two design reference images that served as the visual spec for the login redesign.

### `CLAUDE.md`

Added a **Login Page (`/login`)** section documenting the component-scoped CSS approach, the UI-only sign-up panel, and the `*ngIf` vs `@if` inconsistency — non-obvious patterns that affect future work on this component.

## Not yet implemented

- Sign-up form wiring (requires a `POST /api/auth/register` backend endpoint)
- Social login buttons (no OAuth integration)
- "Forgot Password" button (no backend endpoint)
